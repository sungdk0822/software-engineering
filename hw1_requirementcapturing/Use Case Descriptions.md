## 전체 대여소 리스트 조회 (관리자)

| 전체 대여소 리스트 조회 (관리자) |  |
| --- | --- |
| Actor Action | System Response |
| 1. None | 2. 전체 대여소 목록을 요청할 수 있는 화면 또는 버튼을 표시한다 |
| 3. 전체 대여소 목록 보기 버튼을 클릭한다 | 4. 시스템은 관리자 권한을 확인하고, 등록된 모든 대여소의 리스트를 화면에 표시한다. 각 항목은 대여소 이름, 위치, 수용 가능 자전거 수 등의 정보를 포함한다 |

---

## 대여소 등록

| 대여소 등록 |  |
| --- | --- |
| Actor Action | System Response |
| 1. None | 2. 대여소 등록을 위한 입력 폼을 표시한다 (이름, 위치, 수용 가능 자전거 수 등) |
| 3. 대여소 정보를 입력하고 등록 버튼을 클릭한다 | 4. 입력된 정보의 유효성을 검사한 뒤, 등록된 대여소를 시스템에 저장하고 등록 완료 메시지를 표시한다 |

---

## 관리자 대여소 상세정보 조회

| 관리자 대여소 상세정보 조회 |  |
| --- | --- |
| Actor Action | System Response |
| 1. 전체 대여소 목록을 조회한다 | 2. 대여소 리스트를 표시한다 |
| 3. 특정 대여소 항목을 클릭한다 | 4. 선택한 대여소 정보가 존재하면, 시스템은 해당 대여소의 등록 정보(대여소 이름, 위치(도시, 주소), 자전거 보관 가능 수량, 운영 시간)를 상세히 출력한다.
선택한 대여소 정보가 삭제되었거나 존재하지 않으면, “해당 대여소 정보를 찾을 수 없습니다”라는 메시지를 출력한다. |

## 회원 대여소 상세정보 조회

| 회원 대여소 상세정보 조회 |  |  |
| --- | --- | --- |
| Actor Action | System Response | External Actor Action |
| 1. 조건 검색 또는 리스트를 통해 대여소 목록을 조회한다 | 2. 회원에게 검색 조건에 해당하는 대여소 리스트를 표시한다 |  |
| 3. 특정 대여소를 선택한다 | 4. 해당 대여소의 위치, 운영 시간, 이용 가능 자전거 수 등의 정보를 표시하고, 즉시 대여 및 예약 대기 기능도 제공한다 |  |
| 5. 회원이 즉시 대여 버튼을 클릭한다 | 6. 시스템은 대여 가능한 자전거를 확인하고, 대여 처리 후 완료 메세지와 대여 정보를 표시한다 | 7. 문자 알림 시스템은 ‘즉시 대여 완료’ 알림을 전송한다 |
| 8. 회원이 예약 대기 버튼을 클릭한다 | 9. 시스템은 예약 대기 가능 여부를 확인하고, 예약 대기 등록 후 확인 메세지를 표시한다 | 10. 문자 알림 시스템은 ‘예약 대기 신청 완료’ 알림을 전송한다 |

## 자전거 리스트 조회

| 자전거 리스트 조회 |  |
| --- | --- |
| Actor Action | System Response |
| 1. 관리자가 자전거 리스트 조회 버튼을 클릭한다. | 2. 등록된 모든 자전거 정보를 조회하여 리스트로 출력한다. |

---

## 자전거 등록

| 자전거 등록 |  |
| --- | --- |
| Actor Action | System Response |
| 1. 관리자가 ‘자전거 등록’ 버튼을 클릭한다. | 2. 자전거 등록 입력 폼을 출력한다. |
| 3. 관리자가 자전거 ID, 자전거 제품명, 유형, 소속 대여소, 상태를 입력한다.
’등록’ 버튼을 클릭한다. | 4. 입력된 정보의 유효성을 검사한 뒤, 등록된 자전거를 시스템에 저장하고 등록 완료 메시지를 표시한다. |

---

## 대여중인 자전거 조회

| 대여중인 자전거 조회 |  |  |  |
| --- | --- | --- | --- |
| Actor Action (회원) | System Response | Actor Action (이메일 전송 시스템) | Actor Action (식당 예약 서비스) |
| 1. 회원이 ‘대여중인 자전거 조회’ 메뉴를 클릭한다. | 2. 현재 사용자의 대여 중 자전거 목록을 조회한다. |  |  |
| Extentions |  |  |  |
| 2-1. 회원이 특정 자전거의 ‘반납’ 버튼을 클릭한다. | 2-2. 자전거 반납 처리 후 완료 메시지를 출력한다. |  |  |
| 2-2-1-1. None | 2-2-1-2. 예약 대기자가 존재할 경우 예약 대기 1순위 회원으로의 이메일을 전송을 요청한다. | 2-2-1-3. 이메일을 전송한다. |  |
| 2-2-2-1. 회원이 ‘근처 식당 추천’ 버튼을 클릭한다. | 2-2-2-2. 식당 예약 서비스와 연결한다. |  | 2-2-2-3. 사용자 위치 정보를 기반으로 근처 식당을 추천하고, 예약 기능을 제공한다. |

## 자전거 상세내용 보기

| 대여중인 자전거 조회 |  |
| --- | --- |
| Actor Action | System Response |
| 1. 관리자가 ‘자전거 리스트 조회’ 화면에서 특정 자전거 항목을 클릭한다. | 2. 해당 자전거의 상세 정보를 출력한다. |
| 2-1. 관리자가 ‘삭제’ 버튼을 클릭한다. | 2-2. 자전거 정보를 삭제하고 완료 메시지를 출력한다. |

## 자전거 예약대기 정보 조회

| 대여중인 자전거 조회 |  |
| --- | --- |
| Actor Action | System Response |
